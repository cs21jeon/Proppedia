# 블로그 글쓰기 스타일 가이드 (행복한토끼 / propnet)

> 이 문서는 15개의 실제 블로그 글을 분석하여 작성된 문체 가이드입니다.
> 블로그 글 작성 시 이 가이드를 참고하여 동일한 톤과 스타일을 유지합니다.

---

## 1. 전체 톤 & 말투

- **해요체/합니다체 혼용** - "~입니다", "~합니다"를 기본으로 하되, 중간중간 "~네요", "~거든요", "~데요" 같은 부드러운 종결어미를 섞는다.
- **친근하고 솔직한 톤** - 전문가가 가르치는 느낌이 아니라, "나도 처음엔 몰랐는데 해보니 이렇더라"는 동료의 느낌.
- **겸손한 태도** - "정답이 아닐 수 있습니다", "틀릴 수도 있지만", "혹시 더 쉬운 방법이 있는데 어렵게 한 게 아닌가 싶기도 하네요"
- **가끔 유머/위트** - "한화팬 쯤 되어야 견딜 수 있을 듯합니다", "캡처에 성공하셨습니다. 축하합니다.???"

---

## 2. 문장 구조 & 길이

### 핵심: 짧다. 매우 짧다.

- **한 문장은 보통 1줄, 길어도 2줄**
- **문장 사이에 빈 줄(줄바꿈)을 넣어 호흡을 끊는다**
- 한 단락에 1~3문장만 넣는다

### 예시 (실제 글에서 발췌)

```
쉽지는 않았습니다.

그 과정을 소개해 볼까 합니다.
```

```
알고 나면 아무것도 아닌데

모르면 한없이 어려운 것들 많죠?
```

```
바빠졌습니다. 업무를 시작했습니다.

긴 백수기간을 끝내고 이제 일을 시작합니다.
```

### 절대 하지 않는 것
- 한 문단에 5줄 이상 이어쓰기
- 접속사로 길게 연결된 복문
- 교과서 같은 설명체

---

## 3. 글 구성 (구조)

### 전형적인 구조

1. **도입 (2~5줄)**: 개인 근황, 감상, 날씨 등 가벼운 이야기로 시작
2. **목차/순서 제시** (선택): 번호 리스트로 전체 흐름 안내
3. **본론**: 짧은 문장 + 이미지 설명, 중간중간 개인 의견 삽입
4. **마무리 (2~4줄)**: 짧은 총평 또는 개인 감상
5. **해시태그**: 글 마지막에 관련 태그 나열

### 도입부 패턴 (실제 예시)

- 근황형: "일하면서 공부하는 게 만만치가 않습니다."
- 날씨형: "27도가 넘는 무더위가 비온 뒤 찾아왔습니다."
- 경험형: "공인중개사 공부 시작하면서 온라인쇼핑몰 진행하던 걸 중단했습니다."
- 질문형: "미금시 들어보셨나요?"
- 감탄형: "날이 많이 풀렸습니다. 아침엔 공기가 차게 느껴지더니 낮이 되니 봄 느낌이 납니다."

---

## 4. 자주 쓰는 표현

### 서술/설명
- "~인 것 같습니다" / "~인 것 같네요"
- "~해 보입니다" / "~로 보입니다"
- "~하겠습니다" / "~되겠습니다"
- "~라고 합니다" / "~라고 하네요"
- "~라 하겠습니다"

### 감상/의견
- "~이지 않을까요?"
- "기회일까요? 위기일까요?"
- "괜찮은 지역, 괜찮은 가격 — 이건 언제나 큰 숙제입니다."
- "사연 없는 부동산은 없습니다."
- "판단이 중요한 시기입니다."
- "과연 잘 마무리될 수 있을까요?"

### 전환/연결
- "이제 ~로 갑니다" / "다음 장소로 갑니다"
- "그 과정을 소개해 볼까 합니다"
- "바로 사용법 들어가 보겠습니다"
- "찬찬히 따라와 보시죠. ^^"
- "성공하면 (또는 실패해도) 과정을 공유해 보겠습니다."

### 겸손/솔직
- "쉽지는 않았습니다" / "만만치가 않습니다"
- "정답이 아닐 수 있습니다"
- "분명 숫자를 누락하거나 오류가 있을 수 있죠"
- "방법이 이렇다는 것만 참고하시고"
- "조금이나마 도움이 되셨으면 좋겠다 싶어서"

### 마무리
- "~을 알아봤습니다"
- "~해 보겠습니다" (다음 글 예고)
- "열심히 일해야겠습니다. ^^"
- "그렇지만 이렇게 보람찰 수 없습니다."

---

## 5. 이모티콘/특수문자 사용

- **^^** : 가끔 문장 끝에 사용 (과하지 않게)
- **???** : 의문 강조 시 ("캡처에 성공하셨습니다. 축하합니다.???")
- **ㅜㅜ** : 아쉬움 표현 시 (드물게)
- 이모지는 거의 사용하지 않음 (네이버 블로그 스타일)

---

## 6. 수사적 질문 (핵심 특징)

중간중간 독자에게 질문을 던지는 스타일이 매우 빈번함:

- "검단신도시로 연장되면 이동인구가 더 많아지겠죠?"
- "도대체 노선이 몇 개인가요."
- "이쯤되면 그런데도 10억 밖에? 라는 생각까지 듭니다."
- "이 빌라는 어떤 조직이 만든 빌라일까요?"
- "아파트 이름 누가 짓는걸까요? 어려울수록 비싸진다는 착각일까요?"
- "세 장이면 하겠는데... 30장이라면???"

## 7. 개인 경험/감정 표현

- 어려움을 솔직하게 표현: "n8n Assistant와 싸워가며? 코드를 작성했습니다."
- 포기하려던 경험 공유: "포기할까도 했는데 결국 만들어내긴 했습니다."
- 체감 표현: "지역에 왔을 때의 에너지 같은 게 느껴진달까요? 막연하지만 무시할 수 없는 느낌입니다."
- 음식/보상 언급: "추운 날씨에 고생한 몸을 위해 보신을 합니다", "딱 어울리는 치맥입니다"

---

## 8. 기술/가이드 글 작성 패턴

단계별 설명 시:
1. 번호를 매긴다 (1, 2, 3...)
2. 각 단계는 1~2문장으로 설명
3. 스크린샷/이미지 삽입 위치 표시
4. "~를 누릅니다", "~로 들어갑니다" 같은 지시형 문장
5. 중간에 팁이나 주의사항 삽입: "단, 주의할 점:", "참고로~"

### 예시
```
사진을 눌러서 배너로 사용할 이미지를 선택합니다.

(스크린샷)

배너의 폭이 170pxl이므로

아래의 순서대로 폭을 기입해 줍니다.
```

## 9. 해시태그 규칙

- 글 마지막에 작성
- 밑줄(_) 구분 또는 붙여쓰기
- 핵심 키워드 + 브랜드 태그 조합
- 보통 7~15개
- 예: `#건축물대장 #건축물대장조회 #부동산앱 #부동산백과 #프롭넷`

---

## 10. 하지 말아야 할 것

1. **교과서 같은 딱딱한 설명문** 쓰지 않기
2. **"~하겠습니다" 반복 남발** (로봇 같은 톤) 피하기
3. **한 문단에 5줄 이상** 쓰지 않기
4. **이모지 남발** 하지 않기 (^^, ㅜㅜ 정도만)
5. **"~해 드리겠습니다", "~알려 드리겠습니다"** 같은 과도한 존대 피하기
6. **목차에 이모지(📋, 💡 등)** 넣지 않기 — 텍스트로만 구성
7. **"핵심 정리" 같은 요약 박스** 과하게 쓰지 않기 — 자연스러운 마무리 선호

## 11. 네이버 블로그 포맷 규칙 (마크다운 → 블로그 변환)

> 네이버 스마트에디터는 마크다운을 지원하지 않는다.
> 마크다운 문법을 그대로 입력하면 리스트 기호가 이중으로 표시되는 등 깨짐이 발생한다.
> 따라서 블로그 글 작성 시 아래 규칙을 반드시 따른다.

### 절대 사용하지 않는 마크다운 문법

1. `- 항목` (불릿 리스트) → 네이버 에디터가 자체 불릿을 추가해서 이중 표시됨
2. `1. 항목` (번호 리스트) → 에디터 자체 번호와 겹침
3. `| 표 |` (테이블) → 텍스트로 깨져서 출력됨
4. `**굵게**` (볼드) → 플레인 텍스트로 별표가 그대로 노출됨
5. `[링크](url)` → 텍스트로 노출됨
6. `#`, `##`, `###` (헤딩) → 그대로 노출됨
7. 이모지 (📋, 💡, 👉, 📸, 📱 등) → 사용하지 않음

### 대체 표현법

```
❌ 마크다운 리스트
- 재산세: 토지 보유에 대한 세금
- 종합부동산세: 일정 금액 이상

✅ 자연스러운 문장
재산세는 토지를 보유하면 매년 내는 세금입니다.

종합부동산세는 일정 금액 이상 부동산을 갖고 있을 때 부과됩니다.
```

```
❌ 마크다운 번호 리스트
1. 사이트 접속
2. 주소 입력
3. 조회 버튼 클릭

✅ 문장형 단계 설명
먼저 사이트에 접속합니다.

주소를 입력하고

조회 버튼을 누릅니다.
```

```
❌ 마크다운 테이블
| 구분 | 공시지가 | 실거래가 |

✅ 비교 문장
공시지가는 정부가 정한 가격이고

실거래가는 실제로 거래된 가격입니다.

보통 공시지가가 실거래가의 60~80% 수준이라고 보시면 됩니다.
```

```
❌ 마크다운 헤딩/볼드
## 건축물대장이란?
**건축물대장**은 건물의 신분증입니다.

✅ 플레인 텍스트 소제목
건축물대장이란?

건축물대장은 건물의 신분증이라고 보시면 됩니다.
```

### 링크 처리

```
❌ 마크다운 링크
[부동산백과 앱 다운로드](https://play.google.com/store/...)

✅ URL 직접 표기
부동산백과 앱은 아래 링크에서 다운로드할 수 있습니다.

https://play.google.com/store/apps/details?id=com.proppedia.app
```

### 소제목 표현

네이버 에디터에서는 소제목을 빈 줄로 구분하되 텍스트만 사용한다.

```
(빈 줄)
건축물대장 조회 방법
(빈 줄)
먼저 사이트에 접속합니다.
```

## 12. 글 작성 체크리스트

- [ ] 도입부에 개인 경험/근황/감상이 있는가?
- [ ] 문장이 짧고 호흡이 끊어져 있는가? (1~2줄씩)
- [ ] 중간에 수사적 질문이 1~2개 있는가?
- [ ] 솔직하고 겸손한 표현이 들어가 있는가?
- [ ] 마무리가 자연스러운가? (요약 박스 대신 짧은 감상)
- [ ] 해시태그가 마지막에 있는가?
- [ ] 전체 톤이 "옆자리 동료가 알려주는" 느낌인가?
- [ ] 마크다운 문법(-, 1., **, ##, 테이블, 이모지)이 없는가?
- [ ] 리스트 대신 자연스러운 문장으로 서술되었는가?
- [ ] 링크는 URL을 직접 표기했는가?
